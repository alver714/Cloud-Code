# Cloud Code — Telegram-бот для агентного программирования в облаке

Пишешь промпт в Telegram → на GCP VM полностью автономно работает **Claude Code** или **Codex** в склонированном GitHub-репозитории, а весь ход работы (текст агента, вызовы инструментов, ошибки) стримится обратно в чат.

- **Каждый топик форум-группы = отдельная сессия.** Переключение топиков ничего не прерывает: сессии работают параллельно, у каждой свой workspace и свой контекст разговора.
- **Многоходовость.** Каждый следующий промпт в топике продолжает тот же разговор (`claude --resume` / `codex exec resume`).
- **Полная автономия.** Движки запускаются с `--dangerously-skip-permissions` / `--dangerously-bypass-approvals-and-sandbox` — агент сам запускает команды, правит файлы, коммитит и открывает PR через авторизованный `gh`.

## Архитектура

```
Telegram (форум-группа, long polling)
   │
grammY bot (Node 22+, systemd на GCP VM)
   │
SessionManager (топик → сессия) ──► sessions.json (атомарная запись + .bak)
   │   очередь промптов, глобальный лимит параллельных запусков
   │
Engine (AsyncIterable<AgentEvent>)
   ├── claude -p --verbose --output-format stream-json --dangerously-skip-permissions [--resume id]
   └── codex exec [resume id] --json --dangerously-bypass-approvals-and-sandbox
   │
Workspace: ~/workspaces/<owner>-<repo>-<topicId>/
```

Прогресс показывается «живым» сообщением, которое редактируется раз в ~3.5 с (лимиты Telegram); когда лог перерастает 4096 символов — сообщение замораживается и продолжается новым. Финальный ответ агента приходит отдельными сообщениями с код-блоками.

## Подготовка (один раз)

1. **Бот**: у [@BotFather](https://t.me/BotFather) → `/newbot` → сохрани токен.
2. **Группа**: создай супергруппу, в настройках включи **Topics**, добавь бота **админом** с правом **Manage Topics**.
3. **Свой user id**: напиши что-нибудь [@userinfobot](https://t.me/userinfobot).

## Локальный запуск (для разработки)

```bash
npm install
cp .env.example .env   # заполни TELEGRAM_BOT_TOKEN и ALLOWED_USER_IDS
npm run dev
```

На macOS `CLAUDE_CODE_OAUTH_TOKEN` не нужен — Claude Code берёт auth из Keychain. Codex — из `~/.codex/auth.json`, gh — из своего логина.

## Деплой на GCP (24/7)

Нужен авторизованный `gcloud` на этой машине. Скрипты можно перезапускать без вреда, параметры через env: `VM_NAME`, `GCP_PROJECT`, `GCP_ZONE`.

```bash
./deploy/01-create-vm.sh    # e2-standard-2, Ubuntu 24.04, 50GB (~$50/мес)
./deploy/02-install.sh      # node, gh, claude, codex, systemd-юнит
./deploy/03-push-auth.sh    # .env, ~/.codex/auth.json, gh-логин + smoke-тесты
./deploy/04-deploy.sh       # сборка → заливка → restart сервиса
```

Перед `03` сгенерируй Claude-токен локально: `claude setup-token`.

Обновление кода → просто `./deploy/04-deploy.sh`.
Логи на VM: `sudo journalctl -u coding-bot -f`; сырые JSONL движков: `~/logs/session-*.jsonl`.

## Команды бота

| Команда | Действие |
|---|---|
| `/new owner/repo [claude\|codex]` | новая сессия: топик + клон репозитория |
| `/repos` | выбрать репозиторий из списка GitHub (inline-кнопки) |
| `/model` | клавиатура моделей обоих движков; внутри движка смена БЕЗ потери контекста, между движками — перенос краткой истории |
| `/engine claude\|codex` | сменить движок (история разговора переносится пересказом) |
| `/verbose` | подробный режим: сырые команды и результаты (по умолчанию — компактный) |
| `/usage` | проценты 5-часового/недельного окон обеих подписок + расход бота за сегодня |
| `/budget 500k\|off` | per-session потолок Token Guard |
| `/stop` | остановить текущий запуск + очистить очередь |
| `/reset` | новый разговор в том же workspace |
| `/status` | состояние сессии + git status |
| `/diff` | текущий git diff |
| `/sessions` | все сессии со статусами |

Обычное сообщение в топике = промпт агенту. Неизвестные `/команды` уходят движку как есть (работают кастомные slash-команды Claude Code, например `/review`).

## Вывод и Token Guard

- **Компактный режим (по умолчанию)**: текст агента + человеческие строки действий («Edited src/… +3 −2», описания команд у Claude, reasoning-заметки у Codex). Каждый элемент — отдельное сообщение **без звука**; уведомление приходит только с финальным итогом. `/verbose` включает сырые команды.
- **Token Guard**: считает «рабочие» токены запуска (свежий input + запись кэша + output + 0.1×чтение кэша). На 100k — тихое предупреждение, на 250k — стоп-кран (контекст сохраняется, продолжить можно следующим промптом). Шаговый предохранитель для Codex (200 шагов). Если 5-часовое окно подписки заполнено ≥90% — бот спросит перед запуском. Пороги — в `.env` (`GUARD_*`), точечно — `/budget`.
- **Экономика по умолчанию**: Claude = `sonnet` + `--effort medium` (лимиты в ~3.3× медленнее, фан-аут субагентов укрощён); тяжёлым задачам — `/model fable`. В статистике запуска виден раздутый контекст и подсказка `/reset`.

## Безопасность

- Бот отвечает **только** user id из `ALLOWED_USER_IDS`; остальные апдейты молча отбрасываются первым middleware.
- Токен бота и allowlist вычищаются из окружения движков; Claude-токен не попадает в Codex-процессы и наоборот.
- Полная автономия агентов ограничена VM: это её единственное назначение, важных данных на ней нет.
- Логи движков могут содержать содержимое файлов, которые читал агент — каталог логов создаётся с правами `0700`.
